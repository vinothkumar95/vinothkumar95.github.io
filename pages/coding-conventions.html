Life is a race... Run.... Run....

03 Jun 2018

In the initial days, I will be given a task, most of the times it will be an API to be exposed/modified. It will be exposed/modified in few days. But as I grew matured and start to work with people who consider clean code patterns. I got these comments and observations.

1) If you find something wrong, fix it.
2) If you feel it can be refactored, please go ahead.
3) Add TODO's. TODOs are good if they will be addressed soon.
4) Take code reviews seriously. (Even missing an index on a table can cause your cpu usage to go above 95%. Messing up channels can block the thread.) Code reviews are the best place to learn.

Hope to see you soon!
